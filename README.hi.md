    41|## 🎯 आज की बुद्धिमान विश्लेषण प्रणालियों की समस्या
    42|
    43|वर्तमान AI विश्लेषण प्रणालियाँ — ChatGPT से लेकर एंटरप्राइज़ कोपायलट तक — एक ही मूलभूत दोष साझा करती हैं:
    44|
    45|- ❌ **भ्रम को तथ्य मानना** — LLM बिना वास्तविक डेटा के आधार के आत्मविश्वास से आँकड़े, स्रोत और निष्कर्ष गढ़ते हैं। आप सत्य को कल्पना से अलग नहीं कर सकते।
    46|- ❌ **एकल-मॉडल अंधे धब्बे** — एक मॉडल, एक विश्वदृष्टि। कोई जिरह नहीं, कोई प्रतिस्पर्धी चुनौती नहीं, कोई दूसरी राय नहीं। पूर्वाग्रह अनदेखे रह जाते हैं।
    47|- ❌ **ब्लैक बॉक्स तर्क** — आपको उत्तर मिलता है, लेकिन *कैसे*? कोई प्रमाण श्रृंखला नहीं, कोई स्रोत श्रेय नहीं, तर्क की ऑडिट या पुनरुत्पादन का कोई तरीका नहीं।
    48|- ❌ **पुराना ज्ञान, कोई प्रमाण नहीं** — मॉडल समय में जमे प्रशिक्षण डेटा पर निर्भर करते हैं। वे समाचार, बाज़ार या सेंसर से लाइव इंटेलिजेंस प्राप्त नहीं कर सकते — वे *जानने* के बजाय *अनुमान लगाते* हैं।
    49|- ❌ **कोई स्व-सुधार नहीं** — AI आउटपुट फायर-एंड-फॉरगेट हैं। त्रुटियाँ चुपचाप फैलती हैं। कोई समीक्षा लूप नहीं, कोई गुणवत्ता गेट नहीं, कोई पुनरावृत्ति परिशोधन नहीं।
    50|- ❌ **खंडित कार्यप्रवाह** — डेटा संग्रह, विश्लेषण, बहस और रिपोर्टिंग अलग-अलग उपकरणों में रहते हैं। हर हैंडऑफ़ पर संदर्भ खो जाता है।
    51|- ❌ **शून्य पुनरुत्पादनशीलता** — एक ही प्रश्न दो बार चलाएँ, अलग उत्तर मिलें। कोई निर्धारक ट्रेस नहीं, कोई निर्णय लॉग नहीं, कोई जवाबदेही नहीं।

    53|## 💡 明鉴 इसे कैसे हल करता है
    54|
    55|明鉴 अनुमान को **प्रमाण** से, राय को **बहस** से, और ब्लैक बॉक्स को **ट्रेस** से बदलता है:
    56|
    57|- ✅ **प्रमाण-आधारित** — प्रत्येक विश्लेषण 10+ स्रोतों (Google News, Reddit, GitHub, GDELT, X/Twitter, और अधिक) से रीयल-टाइम डेटा पर आधारित है। कोई भ्रम नहीं, कोई मनगढ़ंत बात नहीं।
    58|- ✅ **मल्टी-एजेंट प्रतिस्पर्धी बहस** — GPT, Gemini, Claude, और Grok केवल सहमत नहीं होते — वे एक-दूसरे को **चुनौती** देते हैं। अंधे धब्बे उजागर होते हैं, पूर्वाग्रहों को चुनौती मिलती है।
    59|- ✅ **पूर्ण ऑडिट ट्रेल** — हर कदम दर्ज किया जाता है: परामर्श किए गए स्रोत, किए गए तर्क, लिए गए निर्णय। पूर्णतः पारदर्शी, पूर्णतः पुनरुत्पादनीय।
    60|- ✅ **रीयल-टाइम इंटेलिजेंस** — लाइव डेटा इनजेशन, स्ट्रीमिंग विश्लेषण, और तत्काल अंतर्दृष्टि वितरण। कोई जमा हुआ प्रशिक्षण डेटा नहीं।
    61|- ✅ **स्व-उपचार पाइपलाइन** — Jarvis इंजन अपने आउटपुट की समीक्षा, आलोचना और पुनरावृत्ति करता है जब तक गुणवत्ता थ्रेसहोल्ड पूरे नहीं हो जाते। त्रुटियाँ आप तक पहुँचने से पहले पकड़ी जाती हैं।

## 🧭 明鉴 की स्थिति: AI निर्णय सलाहकार

> **सामान्य उपकरण नहीं, बल्कि समर्पित सलाहकार टीम।**

AI Agent का युग आ चुका है। Orchestration Layer कई sub-agents और उपकरणों को समन्वित करता है, sandbox वातावरण में जटिल कार्यों को स्वायत्त रूप से पूरा करता है — यह एक प्रभावी प्रतिमान साबित हो चुका है।

明鉴 इस आधार पर और आगे बढ़ता है, **निर्णय बुद्धिमत्ता** पर केंद्रित:

### डिज़ाइन दर्शन

| डिज़ाइन सिद्धांत | 明鉴 का अभ्यास |
|-----------------|----------------|
| Orchestration Layer > Base Model | 9-एजेंट रजिस्ट्री, भूमिका अनुसार मॉडल वितरण |
| रीयल-टाइम स्ट्रीमिंग विश्वास बनाती है | बहस राउंड-दर-राउंड रेंडर, उपयोगकर्ता हर चरण देखता है |
| बहु-उपकरण सहयोग | 12 डेटा स्रोत + बहस इंजन + सिमुलेशन इंजन |
| विफलता पर पुनर्योजन | बहस विफल होने पर स्वचालित रणनीति समायोजन |

### MoE आर्किटेक्चर विचार

明鉴 की 9-एजेंट प्रणाली **Mixture of Experts (MoE)** के मूल विचार से प्रेरित है:

```
उपयोगकर्ता प्रश्न → राउटर (बहस प्रवाह) → विशेषज्ञ संयोजन चयन → स्वतंत्र तर्क → भारित निर्णय
```

जैसे DeepSeek-V3 256 विशेषज्ञों में से कुछ ही सक्रिय करता है, 明鉴 9 एजेंटों में से प्रश्न प्रकार के अनुसार सबसे उपयुक्त संयोजन चुनता है। मुख्य 3 भूमिकाएँ (समर्थक/चुनौतीकर्ता/मध्यस्थ) हमेशा सक्रिय, परिप्रेक्ष्य 6 भूमिकाएँ आवश्यकतानुसार — यह सॉफ्टवेयर स्तर पर स्पार्स एक्टिवेशन है।

   193|#### वातावरण चर
   194|
   195|प्रोजेक्ट रूट में एक `.env` फ़ाइल बनाएँ जिसमें निम्नलिखित चर हों:
   196|
   197|```bash
   198|# ═══════════════════════════════════════════════════════════════
   199|# AI Model Configuration
   200|# ═══════════════════════════════════════════════════════════════
   201|# You only need ONE API key to get started.
   202|# The system automatically uses the same key for all 7 model slots
   203|# (primary, extraction, x_search, report, debate_advocate,
   204|#  debate_challenger, debate_arbitrator) unless you override them.
   205|
   206|PLANAGENT_OPENAI_API_KEY=your_api_key_here
   207|
   208|# Override individual targets (all fall back to shared if unset)
   209|# PLANAGENT_OPENAI_PRIMARY_MODEL=gpt-4.1
   210|# PLANAGENT_OPENAI_PRIMARY_API_KEY=sk-...
   211|# PLANAGENT_OPENAI_EXTRACTION_MODEL=gpt-4.1-mini
   212|# PLANAGENT_OPENAI_DEBATE_ADVOCATE_MODEL=claude-sonnet-4-20250514
   213|# PLANAGENT_OPENAI_DEBATE_CHALLENGER_MODEL=gemini-2.5-flash
   214|# PLANAGENT_OPENAI_DEBATE_ARBITRATOR_MODEL=grok-3
   215|
   216|# ═══════════════════════════════════════════════════════════════
   217|# Database (optional — defaults to SQLite for local dev)
   218|# ═══════════════════════════════════════════════════════════════
   219|# PLANAGENT_DATABASE_URL=postgresql+psycopg://planagent:planagent@localhost:5432/planagent
   220|
   221|# ═══════════════════════════════════════════════════════════════
   222|# Redis (optional — for event bus in production)
