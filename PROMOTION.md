## 版本一：GitHub / 技术社区（详细版）

🔥 开源 | 明鉴 MingJian — AI多代理战略决策平台

做了个开源项目，解决一个核心问题：**当前AI分析系统太不靠谱。**

幻觉当事实、黑箱推理、单一模型盲区、知识过时……这些问题你一定遇到过。

**明鉴怎么解决的？**

👁️ **证据锚定** — 不靠训练数据猜，接入10+实时数据源（Google News、Reddit、GDELT、X/Twitter…），每个结论都有出处

⚔️ **多代理对抗辩论** — GPT、Gemini、Claude、Grok 不是简单投票，而是互相挑战、互相纠错，暴露盲点

🔍 **完整审计追踪** — 每一步推理过程都有记录，完全透明、完全可复现

🛠️ **自愈流水线** — Jarvis引擎自动审查自身输出，迭代到质量达标才交付

**技术栈：** FastAPI + Next.js + SQLAlchemy + Redis Streams + pgvector

支持企业（市场分析、竞争情报）和军事（作战规划、物流）双领域。

⭐ 如果觉得有价值，帮忙点个 Star 鼓励一下：
👉 https://github.com/dashitongzhi/MingJian

## 版本二：掘金 / 知乎 / V2EX（技术深度版）

**我做了一个让多个AI互相"吵架"的决策平台，开源了**

用AI做分析，最头疼的是什么？**你不知道它说的对不对。**

一个模型说"市场前景乐观"，你信吗？它可能在幻觉。你让它再分析一遍，结果又不一样。没有任何审计追踪，出了事找不到原因。

为了解决这个问题，我做了一个开源平台——**明鉴 (MingJian)**：

**核心思路：让多个AI互相挑战，而不是只听一个AI的。**

- 不是简单的"投票表决"，而是真正的对抗性辩论——GPT、Gemini、Claude、Grok各执一词，互相质疑对方的证据和推理
- 所有分析都基于实时数据源，不是靠模型的"记忆"
- 每一步推理都有确定性追踪，可以审计、可以复现
- 内置自愈引擎，输出质量不达标会自动迭代修复

支持企业级（市场分析、供应链）和军事级（作战模拟、后勤）双领域场景。

**技术栈：** Python FastAPI + Next.js 15 + SQLAlchemy + Redis Streams + pgvector + 多模型并行调度

项目完全开源，MIT协议，欢迎Star和贡献：
👉 https://github.com/dashitongzhi/MingJian
